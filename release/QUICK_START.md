# 快速开始指南

> 5分钟快速上手文件重复校验工具

---

## 🚀 两种使用方式

### 方式一：直接运行（需要Python环境）

**适合人群**: 开发者、有Python环境的用户

```bash
# 1. 克隆项目
git clone https://github.com/Xxy15021337046/duplicate-file-finder.git
cd duplicate-file-finder

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动程序
python run_gui.py
```

✅ **优点**: 
- 启动速度快
- 可随时修改代码
- 占用空间小

❌ **缺点**: 
- 需要安装Python
- 需要配置环境

---

### 方式二：打包成exe（无需Python环境）

**适合人群**: 普通用户、无Python环境的用户

#### Windows用户

1. **双击运行** `build.bat`
2. 等待打包完成（约1-2分钟）
3. 在 `dist` 文件夹中找到 `文件重复校验工具.exe`
4. 双击exe文件即可使用

#### Linux/macOS用户

```bash
# 1. 赋予执行权限
chmod +x build.sh

# 2. 运行打包脚本
./build.sh

# 3. 在 dist 文件夹中找到可执行文件
ls dist/
```

✅ **优点**: 
- 无需安装Python
- 双击即可运行
- 可复制到任意电脑使用

 **缺点**: 
- 首次打包需要时间
- 文件体积较大（100-200MB）
- 首次启动较慢

## 📖 使用教程

### 第一步：添加扫描目录

1. 点击 **"添加目录"** 按钮
2. 选择要扫描的文件夹
3. 可添加多个目录进行交叉比对

### 第二步：选择检测模式

| 模式 | 适用场景 | 推荐设置 |
|------|---------|---------|
| **精确匹配** | 完全相同的文件 | 默认设置即可 |
| **图片相似度** | 相似照片去重 | 阈值12，精确模式 |
| **视频相似度** | 相似视频去重 | 阈值0.7，最小时长10秒 |

### 第三步：开始检测

1. 点击 **"开始检测"** 按钮
2. 等待进度条完成
3. 查看日志了解扫描进度

### 第四步：查看结果

1. 点击 **"结果"** 按钮
2. 双击组别查看详情
3. 使用筛选功能快速定位

## 💡 实用技巧

### 技巧1：增量扫描

**场景**: 定期清理新增文件

1. 首次扫描使用"全量扫描"
2. 后续扫描使用"增量扫描"
3. 系统自动跳过已处理文件

### 技巧2：历史数据恢复

**场景**: 重启后查看上次结果

1. 重启程序后直接点击"结果"
2. 系统自动从数据库加载
3. 无需重新扫描

### 技巧3：快速定位大文件

**场景**: 优先清理占用空间大的文件

1. 点击"总大小"列标题
2. 按文件大小降序排序
3. 优先处理大文件组

### 技巧4：批量隐藏合法相似

**场景**: 图标不同尺寸版本等合法相似

1. 右键点击组别
2. 选择"隐藏此组"
3. 该组不再显示

## ❓ 常见问题速查

### Q: 扫描速度太慢？

**A**: 
- 使用SSD硬盘（最快提升）
- 减少并行线程数（HDD用4-8）
- 使用增量扫描

### Q: 检测结果不准确？

**A**:
- 精确匹配：检查文件是否真的相同
- 图片相似度：调整阈值到12-15
- 视频相似度：降低阈值到0.6-0.65

### Q: 如何导出结果？

**A**: 
- 当前版本暂不支持直接导出
- 可截图保存结果窗口
- 关注后续版本更新

### Q: 支持哪些格式？

**A**:
- 精确匹配：所有格式
- 图片：JPG/PNG/GIF/BMP/WebP
- 视频：MP4/AVI/MKV/MOV/WMV等20+种

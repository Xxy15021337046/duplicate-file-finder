# 高性能重复文件检测系统

针对TB级别大规模数据优化的重复文件检测工具，使用SQLite3存储索引，支持并行处理和多级哈希校验。提供图形界面（GUI）和命令行两种使用方式。

## 核心特性

### 精确匹配（Exact Match）
- **高性能**: 多级哈希策略（文件大小 → 部分哈希 → 完整哈希）快速筛选
- **大规模支持**: 专为TB级别数据设计，优化内存使用和数据库性能
- **并行处理**: 多线程并发计算哈希值，自动检测CPU核心数并动态调整线程数
- **增量扫描**: 基于SQLite3的持久化存储，支持断点续扫
- **精确匹配**: MD5完整文件哈希确保100%准确性
- **JSON输出**: 结构化结果便于后续处理
- **智能过滤**: 支持文件类型、后缀、大小等多维度筛选
- **实时交互**: 点击排序、文件跳转、右键隐藏等便捷功能

### 相似度检测（Similarity Detection）✨ NEW
- **多算法融合**: pHash + dHash + 颜色直方图，准确率高达99%+
- **三级过滤**: 感知哈希 → 差异哈希 → 直方图比对，层层筛选
- **可视化预览**: 点击文件即可预览图片，无需打开外部程序
- **历史恢复**: 重启后自动加载历史检测结果，无需重新添加目录
- **灵活配置**: 相似度阈值、检测模式、扫描方式可调
- **列排序**: 点击列标题按相似度、数量、大小排序
- **懒加载缓存**: 缩略图按需加载并缓存，节省内存提升性能

## 技术架构

### 三级过滤策略

1. **文件大小过滤**: 首先按文件大小分组，快速排除唯一大小的文件
2. **部分哈希过滤**: 计算前1MB的MD5哈希，进一步缩小候选范围
3. **完整哈希验证**: 对剩余文件计算完整MD5哈希，确认重复

### 性能优化

- **64KB读取块**: 优化I/O性能，提升哈希计算速度
- **批量插入**: 每5000条记录批量提交，减少数据库事务开销
- **动态线程池**: 自动检测CPU核心数×2，最多16线程
- **SQLite3 WAL模式**: 提高并发写入性能
- **内存映射**: 256MB mmap加速数据访问
- **流式读取**: 分块读取大文件，控制内存占用

### 命令行模式

#### 基本用法

```bash
# 扫描单个目录
python duplicate_finder.py /path/to/directory

# 扫描多个目录
python duplicate_finder.py /dir1 /dir2 /dir3
```

#### 高级选项

```bash
# 自定义数据库和输出文件
python duplicate_finder.py /data --db custom.db --output results.json

# 调整并行线程数（根据CPU核心数自动配置）
python duplicate_finder.py /data --workers 16

# 自定义读取块大小（针对大文件优化）
python duplicate_finder.py /data --chunk-size 65536

# 指定文件类型过滤
python duplicate_finder.py /data --ext-filter jpg png mp4
```

#### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| directories | 要扫描的目录路径（必填，可多个） | - |
| --db | SQLite数据库文件路径 | file_index.db |
| --output | 输出JSON文件路径 | duplicates.json |
| --workers | 并行工作线程数（自动检测CPU核心数×2） | 8 |
| --chunk-size | 文件读取块大小（字节） | 65536 |
| --ext-filter | 允许的文件后缀（可多个，不加"."） | 无过滤 |

## 性能参考

在典型配置下的性能表现：

- **扫描速度**: 约1000-5000文件/秒（取决于文件大小和存储介质）
- **内存占用**: 约500MB-2GB（取决于并发线程数和批量大小）
- **数据库大小**: 约每百万文件100-200MB

### 优化建议

**SSD存储**:
- 使用NVMe SSD可获得3-5倍性能提升

**线程数配置**:
- HDD: 4-8线程
- SSD: 8-16线程
- NVMe: 16-32线程（自动检测并配置）

**针对大文件**:
- 增加 `--chunk-size` 到 65536 或更大（默认已是64KB）
- 减少 `--workers` 避免I/O竞争

**针对小文件**:
- 保持默认 `--chunk-size`
- 增加 `--workers` 提高并发

## 工作原理

```
开始扫描
  ↓
遍历目录树 → 收集文件信息 → 应用文件类型过滤 → 批量存入SQLite
  ↓
按文件大小分组 → 筛选出有大小的文件
  ↓
计算部分哈希(前1MB) → 并行处理 → 筛选出相同部分哈希
  ↓
计算完整哈希 → 并行处理 → 确认重复文件
  ↓
导出JSON报告
  ↓
GUI展示结果 → 支持筛选、排序、跳转等操作
```

## 注意事项

1. **首次扫描**: 会创建数据库文件，后续扫描可复用
2. **权限问题**: 确保有读取目标目录的权限
3. **符号链接**: 自动跳过符号链接文件
4. **隐藏文件**: 自动跳过以`.`开头的隐藏目录
5. **中断恢复**: 可随时中断，已扫描的数据保存在数据库中
6. **每次扫描**: 默认清空旧数据，确保结果准确
7. **编码问题**: Windows终端使用GBK编码，可能无法显示特殊字符

## 常见问题

### Q: 为什么步骤一和步骤二的文件数量不一致？
A: 步骤一显示的是"重复组数"，步骤二显示的是"重复文件总数"。例如：3个相同的文件算作1个组，但包含3个文件。

### Q: Python线程能利用多核吗？
A: Python的GIL限制了CPU密集型任务的多核使用，但本工具是I/O密集型（文件读取、哈希计算），可以有效利用多线程等待时间，实现准并行处理。

### Q: 如何提高扫描速度？
A: 
1. 使用SSD硬盘
2. 增加线程数（`--workers`参数）
3. 使用文件类型过滤减少扫描范围
4. 关闭其他占用磁盘I/O的程序

### Q: 扫描过程中可以中断吗？
A: 可以点击"停止"按钮中断扫描，已扫描的数据会保存在数据库中。下次扫描时会清空旧数据重新开始。
